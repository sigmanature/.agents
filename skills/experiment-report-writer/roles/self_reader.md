# Self Reader Style

Use this style when the document is primarily for the author's own understanding, later reuse, or as a source draft for higher-level reports.

## Tone

- still concise
- more explanatory than leadership-facing versions
- explicitly define terms that can be misunderstood
- allow moderate process detail when it changes interpretation

## Emphasis

Prioritize in this order:

1. what the data says
2. what each metric means
3. what current interpretation is safe versus premature
4. what to change next in experiment design

## Required additions

Always include:

- metric definition section when terms like "window ratio", "cumulative ratio", or similar appear
- current workload/config summary
- explicit statement of what the current test can prove and what it still cannot prove

## Suggested phrasing

Prefer phrases such as:

- "这里先把术语说清楚"
- "当前这个指标应这样理解"
- "这组数据目前只能支持到这个层次的判断"
- "如果要继续提高压力，优先考虑"
