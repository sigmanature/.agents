# 规则 2：查阅参考文献

## 适用范围

本规则规定在毕业论文写作过程中检索、筛选、引用和管理学术文献的标准流程。

## 核心原则

1. **真实可溯源** — 每一处引用都必须能在数据库中查到原始文献
2. **时效性优先** — 优先引用近 5 年内的文献，经典文献除外
3. **权威性保障** — 优先引用顶级期刊、顶会、高被引论文
4. **相关性严格** — 引用必须与论述内容直接相关，禁止堆砌无关文献

---

## 文献检索流程

### 步骤 1：明确检索需求

在检索前必须明确：
- 检索主题（中英文关键词）
- 时间范围（建议近 5 年）
- 文献类型（期刊/会议/综述）
- 所需数量（根据章节需求确定）

### 步骤 2：使用 Google Scholar 检索

#### 基础检索

使用 `gs-search` skill：
```
/gs-search [英文关键词]
```

示例：
```
/gs-search "medical question answering" "large language model"
```

#### 高级检索

使用 `gs-advanced-search` skill，支持以下过滤条件：

| 过滤条件 | URL 参数 | 示例 |
|---------|---------|------|
| 作者 | `as_sauthors` | `as_sauthors="Yoshua Bengio"` |
| 期刊/会议 | `as_publication` | `as_publication=Nature` |
| 起始年份 | `as_ylo` | `as_ylo=2020` |
| 结束年份 | `as_yhi` | `as_yhi=2025` |
| 精确短语 | `as_epq` | `as_epq="chain of thought"` |
| 标题检索 | `as_occt=title` | 仅搜索标题 |

示例：
```
/gs-advanced-search "搜索 2020 年以后 Nature 上关于 medical AI 的论文"
```

### 步骤 3：结果筛选与评估

对检索结果按以下标准筛选：

| 评估维度 | 优秀标准 | 可接受标准 | 排除标准 |
|---------|---------|-----------|---------|
| 相关性 | 与论述主题直接相关 | 间接相关但提供背景 | 明显无关 |
| 时效性 | 近 3 年内发表 | 近 5 年内发表 | 超过 10 年（非经典） |
| 权威性 | 顶刊/顶会（Q1/CCF-A） | 知名期刊/会议 | 预警期刊/掠夺性期刊 |
| 被引量 | 高被引（>100） | 中等被引（>10） | 零被引且发表多年 |
| 可获取性 | 开放获取或机构订阅 | 可通过 Sci-Hub 获取 | 完全无法获取 |

### 步骤 4：引用追踪

#### 正向追踪（被引文献）

使用 `gs-cited-by` skill 查找引用某篇论文的后续研究：
```
/gs-cited-by [data-cid]
```

用途：了解该研究方向的最新进展。

#### 反向追踪（参考文献）

阅读目标论文的参考文献列表，追溯经典工作和理论基础。

### 步骤 5：文献导出与管理

使用 `gs-export` skill 将文献导出到 Zotero：
```
/gs-export [data-cid1] [data-cid2] ...
```

导出的 BibTeX 条目应包含：
- 完整的作者列表
- 准确的论文标题
- 期刊/会议名称（使用标准缩写）
- 发表年份
- 卷号、期号、页码
- DOI（如有）

## 引用规范

### 引用格式

严格遵循学校规定的引用格式（GB/T 7714 或学校自定义格式）。

### 引用原则

| 原则 | 说明 | 示例 |
|------|------|------|
| 首次出现 | 首次引用需给出完整信息 | Zhang 等 [1] 首次提出了…… |
| 避免堆砌 | 同一观点引用 1-2 篇即可 | 禁止"A 认为 [1]，B 认为 [2]，C 认为 [3]" |
| 原始文献 | 尽量引用原始文献而非综述 | 引用提出该方法的原始论文 |
| 平衡引用 | 兼顾国内外研究 | 不能只引用中文或只引用英文 |
| 时效分布 | 经典+近期结合 | 既有奠基性文献，也有最新进展 |

### 常见引用错误

| 错误类型 | 错误示例 | 正确做法 |
|---------|---------|---------|
| 二次引用 | 某某指出 [1]（实际 [1] 是综述） | 追溯并引用原始文献 |
| 断章取义 | 只引用支持自己观点的部分 | 客观呈现文献观点 |
| 虚假引用 | 编造或篡改引用信息 | 所有引用必须可验证 |
| 过度自引 | 大量引用自己未发表的工作 | 自引比例不超过 10% |

## 文献综述写作规范

### 综述结构

1. **按时间线梳理** — 展示领域发展历程
2. **按方法分类** — 对比不同技术路线
3. **指出现有不足** — 为本文研究定位
4. **小结段落** — 归纳目前该领域研究进展

### 综述写作禁忌

| 禁忌 | 说明 |
|------|------|
| 简单罗列 | 不能只是"A 做了 X，B 做了 Y，C 做了 Z" |
| 缺乏评述 | 必须对现有工作进行分析和评价 |
| 忽略最新进展 | 必须包含近 2-3 年的重要工作 |
| 忽视反面工作 | 也要提及与本文观点不一致的研究 |

---

## CAPTCHA 处理

Google Scholar 可能触发 CAPTCHA 验证：
1. 停止所有自动化操作
2. 提示用户在浏览器中完成人工验证
3. 等待用户确认后再继续
4. 切勿自动重试

---

## 输出要求

所有文献检索和引用审查的结果，必须以表格形式汇报（详见 SKILL.md 输出格式要求）。
